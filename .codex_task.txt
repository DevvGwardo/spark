## GOAL
Fix tool invocation persistence in chat mode AND make plan mode useful by allowing read-only + file-creation tools.

## Context

Two bugs/issues:

### Issue 1: Tool Results Lost on Persistence
During streaming, tool results show fine (AI SDK's in-memory state). But when `onFinish` fires in `server/routes/chat.ts` (around line 777), it calls `persistAssistantSnapshot` which relies on the AI SDK's `message` parameter having complete `parts`/`toolInvocations`. If the `data` stream protocol doesn't populate those fields for server-side tool execution, the persisted message ends up with empty tool invocations.

There IS a fallback (`toolActivityRef` / `serverToolEventsRef`) but it only works for Hermes providers with custom SSE events — not for standard providers.

The rendering fallback in `src/components/chat/ChatArea.tsx` around line 706-712:
- Tries `partsToolInvocations OR msg.toolInvocations` → if both empty → nothing renders

Result: you see tool calls happen during streaming, but after page reload or conversation switch, they vanish.

**Fix needed:** Ensure `persistAssistantSnapshot` captures tool invocations from the streaming state (`toolActivityRef` data) even when the AI SDK's `message` object doesn't include them. Make the fallback work for all providers, not just Hermes.

### Issue 2: Plan Mode Over-Restricts
The server has a `PLAN_MODE_SYSTEM_PROMPT` at `server/routes/chat.ts:38-53` that tells the model:
- "You MAY NOT: write files, edit files, delete files, run mutating commands"
- Lines 602-606 filter out mutating tools (`write_file`, `execute_python`, `run_command`) from the available tools list

Plan mode is read-only by design, but it's TOO restrictive. Users want plan mode to be able to read code AND create plan documents. The key tools that should work in plan mode:
- `read_repo_file` (read-only, already works)
- `list_repo_files` (read-only, already works)
- `create_html_file` (file creation for plan documents)
- `grep_repo` (read-only search)

Tools that should stay blocked: `edit_repo_file`, `delete_repo_file`, `run_command`, `execute_python`

**Fix needed:** Update the filtered tools list at lines 602-606 to allow `create_html_file` (and similar non-destructive file creation tools) while keeping truly mutating tools blocked.

### Issue 3: Verify Rendering
Make sure the `toolInvocations` extraction in `src/components/chat/ChatArea.tsx` handles ALL cases where data exists — including when `parts` array has tool invocations vs when `toolInvocations` is a top-level property.
